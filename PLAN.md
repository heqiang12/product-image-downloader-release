# 京东图片批量下载工具开发计划

## 1. 项目定位

本项目用于开发一个面向电商运营人员的京东商品素材批量下载工具。

第一阶段只支持京东商品链接，优先把基础能力做完整、稳定、可继续扩展。工具需要让用户输入一个或多个京东商品链接后，自动解析商品素材，并按商品分类保存主图、详情图、SKU 图等图片资源。

## 2. 第一版目标

### 2.1 核心目标

- 支持京东商品详情页链接解析。
- 支持单链接下载。
- 支持多链接批量下载。
- 支持下载主图、详情图、SKU 图。
- 支持图片去重。
- 支持下载进度展示。
- 支持失败重试。
- 支持按商品自动创建文件夹。
- 支持下载结果日志和元数据记录。
- 支持用户选择保存目录。

### 2.2 暂不纳入第一版

- 淘宝、天猫、1688、拼多多等平台。
- 自动登录京东账号。
- 店铺级批量采集。
- 视频下载。
- 代理池。
- 云端任务。
- 图片压缩、翻译、抠图、水印处理。
- 高强度反爬绕过。

## 3. 推荐技术栈

### 3.1 桌面端方案

推荐使用：

- Electron：桌面应用外壳，方便访问本地文件系统。
- Vue：构建工具界面。
- TypeScript：保证下载任务、解析结果和状态流转的类型安全。
- Element UI 或现有 UI 组件体系：快速构建表单、表格、进度条、弹窗。
- Playwright：打开并解析京东商品页，处理动态加载内容。
- Node.js Stream：稳定下载图片到本地。
- SQLite：记录任务、商品、图片、失败原因和历史下载结果。

### 3.2 为什么选择 Electron + Playwright

京东商品页存在动态加载、懒加载和页面脚本数据。纯 HTTP 请求加 HTML 解析可以处理一部分场景，但详情图、SKU 图和懒加载图片容易遗漏。

Playwright 可以使用真实浏览器环境加载页面，再从 DOM、页面脚本和网络响应中提取资源，第一版稳定性更好。Electron 则适合做本地文件保存、任务管理和用户界面。

## 4. 产品形态

第一版建议做成桌面应用，主界面包含以下区域：

- 链接输入区：支持粘贴多行京东商品链接。
- 任务列表区：展示商品链接、商品标题、状态、进度、成功数、失败数。
- 下载设置区：选择保存目录、下载内容类型、并发数量。
- 操作区：开始下载、暂停任务、重试失败、打开目录。
- 日志区：展示解析失败、下载失败、重试记录。

## 6. 核心数据结构

### 6.1 商品素材结果

```ts
export interface ProductAssets {
  platform: 'jd';
  skuId: string;
  title: string;
  sourceUrl: string;
  images: {
    main: AssetItem[];
    detail: AssetItem[];
    sku: AssetItem[];
  };
}

export interface AssetItem {
  url: string;
  type: 'main' | 'detail' | 'sku';
  filename?: string;
  referer?: string;
}
```

### 6.2 下载任务

```ts
export interface DownloadTask {
  id: string;
  sourceUrl: string;
  skuId?: string;
  title?: string;
  status: 'pending' | 'parsing' | 'downloading' | 'success' | 'failed' | 'paused';
  progress: {
    total: number;
    success: number;
    failed: number;
  };
  errorMessage?: string;
  outputDir?: string;
  createdAt: number;
  updatedAt: number;
}
```

## 7. 京东解析策略

### 7.1 链接识别

从以下形式中提取 SKU ID：

```text
https://item.jd.com/100012043978.html
https://item.m.jd.com/product/100012043978.html
https://item.jd.com/100012043978.html?xxx=yyy
```

第一版只处理商品详情页，不处理搜索页、店铺页、活动页。

### 7.2 图片提取优先级

按以下顺序提取图片：

1. 页面 DOM 中的图片地址。
2. 图片懒加载属性，例如 `data-src`、`data-lazy-img`、`data-original`。
3. 页面内联脚本中的商品配置、图片列表、SKU 图片数据。
4. Playwright 监听到的网络响应中包含的图片 URL 或商品详情数据。

### 7.3 图片 URL 标准化

需要处理：

- `//img10.360buyimg.com/...` 补全为 `https://img10.360buyimg.com/...`。
- 过滤空地址、base64、占位图、低价值图标。
- 去除重复 URL。
- 根据需要将小尺寸规则替换为高清地址规则。
- 保留原始 URL 到 `meta.json`，方便排查。

### 7.4 分类规则

第一版分类可以采用混合策略：

- 主图：商品页主图区域、图片列表或脚本中的主图数组。
- 详情图：商品详情区域、详情 HTML、详情接口返回图片。
- SKU 图：规格选择区域、颜色/尺码规格项关联图片。

如果无法准确分类，则放入 `unknown` 或记录到日志。第一版界面中可以默认只展示 main/detail/sku，内部保留 unknown 方便调试。

## 8. 下载与文件保存

### 8.1 保存结构

每个商品保存到独立目录：

```text
downloads/
└── 商品标题_skuId/
    ├── main/
    ├── detail/
    ├── sku/
    ├── unknown/
    └── meta.json
```

### 8.2 文件命名

建议规则：

```text
main_001.jpg
main_002.jpg
detail_001.jpg
sku_001.jpg
```

同时需要：

- 清理 Windows 非法字符。
- 限制文件名长度。
- 根据响应 `Content-Type` 或 URL 后缀确定扩展名。
- 同名文件自动递增。

### 8.3 下载策略

- 默认并发数：5。
- 单张图片失败重试：2 次。
- 请求超时：30 秒。
- 使用 Referer：商品详情页 URL。
- 保存前校验响应状态码和内容类型。
- 下载完成后写入 `meta.json`。

## 9. 任务状态流转

```text
pending
  ↓
parsing
  ↓
downloading
  ↓
success

任意阶段失败:
failed

用户暂停:
paused
```

失败任务支持重新进入 `pending` 后重试。

## 11. 开发里程碑

### 阶段 1：项目骨架

- 初始化 Electron + Vue + TypeScript 项目。
- 配置 ESLint、Prettier、构建脚本。
- 建立主进程、渲染进程、IPC 通信。
- 建立基础页面布局。

验收标准：

- 应用可以启动。
- 页面可以输入链接。
- 渲染进程可以通过 IPC 调用主进程。

### 阶段 2：京东链接解析器

- 实现 SKU ID 提取。
- 实现 Playwright 页面加载。
- 实现商品标题提取。
- 实现主图提取。
- 实现详情图提取。
- 实现 SKU 图提取。
- 输出统一 `ProductAssets` 结构。

验收标准：

- 输入一个京东商品链接，可以解析出标题和图片列表。
- 至少覆盖主图和详情图。
- 解析结果可以保存为 JSON 供调试。

### 阶段 3：下载核心

- 实现图片 URL 标准化。
- 实现文件名安全处理。
- 实现并发下载。
- 实现失败重试。
- 实现分类目录保存。
- 实现 `meta.json`。

验收标准：

- 单商品图片可以完整下载到本地。
- 文件夹分类清晰。
- 失败图片有明确错误记录。

### 阶段 4：批量任务

- 实现任务队列。
- 支持多链接添加。
- 支持并发任务控制。
- 支持任务状态更新。
- 支持暂停和重试失败。

验收标准：

- 多个京东链接可以排队下载。
- 界面能实时展示每个任务状态和进度。

### 阶段 5：本地持久化

- 接入 SQLite。
- 保存任务历史。
- 保存商品元数据。
- 保存图片下载状态。
- 应用重启后恢复任务列表。

验收标准：

- 关闭应用后再次打开，能看到历史任务。
- 失败任务可以继续重试。

### 阶段 6：打包与交付

- 配置 Windows 打包。
- 处理 Playwright 浏览器依赖。
- 输出安装包或免安装包。
- 编写使用说明。

验收标准：

- 在 Windows 环境可以正常安装或解压运行。
- 普通用户无需命令行即可使用。

### 阶段 5.5：平台抽象重构

- 定义统一 `PlatformAdapter` 接口。
- 将京东 URL 识别、链接标准化、商品解析迁移到 `platforms/jd`。
- 增加平台注册表，统一处理链接识别和解析分发。
- 任务数据结构预留 `platform` 字段。

验收标准：

- 当前京东能力保持可用。
- 主进程不再直接依赖京东 URL 判断。
- 新增平台只需要实现 adapter 并注册。

### 阶段 5.6：登录中心与 Cookie/Profile 持久化

- 为每个平台创建独立浏览器 Profile。
- 支持打开平台登录窗口。
- 支持检测登录状态。
- 解析商品时复用对应平台 Profile。
- 下载请求优先复用登录态 Cookie、Referer、User-Agent。

验收标准：

- 用户可以在工具内登录京东。
- 重启应用后仍可复用登录状态。
- 未登录、登录过期时页面有明确提示。

### 阶段 5.7：Excel 导入与模板导出

- 支持 `.xlsx` 导入商品链接。
- 支持下载 Excel 模板。
- Excel 字段至少包含：名称、链接、平台。
- 平台字段可选，系统可自动识别。
- 导入后生成批量任务。

验收标准：

- 可以从 Excel 一次导入多个商品链接。
- 无效链接和重复链接有明确提示。
- 模板文件可直接给运营人员使用。

### 阶段 5.8：页面重构

- 重构主界面为平台登录区、导入区、任务区、日志/详情区。
- 增加平台选择和登录状态展示。
- 增加 Excel 导入、模板导出。
- 增加任务详情，展示主图、详情图、SKU 图数量和失败原因。
- 优化长链接、长错误信息和窄屏布局。

验收标准：

- 登录、导入、下载、查看结果形成清晰闭环。
- 长链接和错误信息不会撑坏布局。

### 阶段 5.9：京东真实商品专项测试

- 准备普通商品、多 SKU 商品、详情图较多商品等测试样本。

## 13. 风险与应对

### 13.1 页面结构变化

京东页面结构可能变化，导致解析失效。

应对：

- 解析逻辑分层，不只依赖单一 DOM 选择器。
- 保留网络响应解析和脚本数据解析。
- 保存调试 JSON，方便快速定位问题。

### 13.2 反爬限制

频繁请求可能触发验证或限制。

应对：

- 第一版控制并发和请求频率。
- 使用真实浏览器加载页面。
- 下载请求带 Referer 和 User-Agent。
- 明确不做绕过账号安全或验证码的能力。

### 13.3 图片分类不准确

不同商品详情页结构不同，可能导致 SKU 图或详情图分类错误。

应对：

- 先保证图片完整下载。
- 分类规则持续优化。
- 对不确定图片放入 unknown 并写入日志。

### 13.4 打包体积较大

Electron + Playwright 会增加安装包体积。

应对：

- 第一版优先稳定性。
- 后续可评估 Tauri + 外部浏览器协议，或拆分解析服务。

## 14. 第一版验收标准
