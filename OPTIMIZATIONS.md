# 优化记录

这个文件用于记录真实使用中发现的问题和后续优化项，按优先级逐步处理。

## P0 运行环境

- Playwright Chromium 依赖必须安装，否则任务会失败并提示 `Executable doesn't exist`。
  - 当前处理：已执行 `npm run install:browsers`。
  - 当前处理：应用内京东解析已优先改用 Electron 同登录分区隐藏窗口；Playwright 主要作为核心解析器的备用/脚本调试能力。
  - 后续优化：应用启动时检测备用 Chromium 是否可用，缺失时在界面给出明确提示。

## P0 账号安全保护

- 不做验证码绕过、指纹伪装、代理池规避等对抗平台安全策略的能力。
- 当京东页面提示账号安全风险、需要京东商城 APP 验证时，应立即停止当前解析。
  - 当前处理：解析器检测到“账号存在安全风险 / 京东商城 APP / 完成安全验证”等提示后，会停止任务并提示用户先完成安全验证。
- 默认采用保守下载策略。
  - 当前处理：页面提供“安全模式”开关；安全模式使用图片并发 2、请求间隔 800ms；关闭安全模式后可自定义图片并发和请求间隔。
  - 注意：即使关闭安全模式，检测到京东账号安全风险页时仍会停止任务。
  - 后续优化：增加冷却时间提示、批量任务之间的随机等待和已下载商品缓存。

## P1 真实下载稳定性

- 京东解析需要滚动触发懒加载。
  - 现状：进入商品页后等待 `networkidle`，随后直接读取页面内容。
  - 风险：详情图、SKU 图可能未加载，导致图片遗漏。
  - 当前处理：解析前自动滚动页面；检测新版 PC 商品页时会点击“商品详情”标签，再滚动采集详情图。

- 京东新版 PC 商品页素材区域规则。
  - 轮播主图：`.image-carousel img.image`、`#spec-img`、`#spec-n1 img`，兼容旧版 `#spec-list/#preview`。
  - 详情图：点击 `#SPXQ-tab-column` 后，从 `#detail-main/#detail-top/#detail-footer`、`.graphicContent`、`.ssd-module-wrap` 采集。
  - SKU 图：`.specification-item-sku-image`，兼容旧版 `#choose-attrs`。
  - 后续优化：增加离线 HTML 结构分析脚本，用本地样本快速回归选择器规则。

- 下载请求需要携带登录 Cookie。
  - 现状：解析阶段会注入登录 Cookie，下载阶段只带 `Referer` 和 `User-Agent`。
  - 风险：部分图片在登录态或风控场景下解析得到 URL，但 Node 下载失败。
  - 建议：把平台 Cookie 传入下载器，并生成 `Cookie` 请求头。

- Playwright 应复用更完整的登录环境。
  - 现状：Electron 登录窗口使用持久化分区，Playwright 解析阶段只手动注入 Cookie。
  - 风险：localStorage、IndexedDB、设备态等没有进入解析上下文。
  - 当前处理：应用内 JD 任务解析已改用同一个 Electron 持久化分区的隐藏窗口，继承登录窗口的 Cookie、localStorage、IndexedDB 等状态。
  - 后续优化：如果详情图仍缺失，继续检查京东详情接口是否需要额外触发或从接口响应中提取。

- 过滤页面素材和小图标。
  - 当前处理：URL 规范化会过滤 `imagetools`、`babel`、`devfe`、`uba`、`da`、`cc`、`libres`、`retail-mall` 等页面素材路径，并把 `/n0/jfs/` 主图归一到 `/n1/jfs/` 去重。
  - 后续优化：增加最小尺寸/文件大小规则，避免少量店铺图标进入未分类。

## P1 用户操作

- 添加重复链接时需要明确反馈。
  - 当前处理：如果链接已存在，页面会选中已有任务，并提示“任务区已有相同商品”。

- 失败任务需要可删除。
  - 当前处理：任务行的“删除”按钮已放到操作区第一个位置，失败任务也可删除。

## P2 产品能力

- 增加只解析不下载模式。
  - 当前处理：页面可以添加“解析任务”；任务只执行页面解析和图片分类统计，不下载图片。
  - 用途：调试京东主图、详情图、SKU 图规则，减少账号安全风险。

- 增加下载类型选择。
  - 目标：主图、详情图、SKU 图、未知图片可以按需勾选。
  - 当前处理：页面已增加轮播主图、详情图、SKU 图勾选；添加任务和 Excel 导入会把选择写入任务；下载器默认不再下载 `unknown`。
  - 后续优化：根据真实商品样本继续提高主图、详情图、SKU 图分类准确率。

- 增加任务中止/暂停。
  - 当前 `paused` 状态只在类型里存在，队列没有真正的取消控制。
  - 建议：增加 AbortController，并区分暂停队列和取消单任务。

- 增加失败明细展示。
  - 现状：任务区只展示总错误信息。
  - 建议：从 `meta.json` 或任务结果中展示失败图片 URL、HTTP 状态和重试次数。

## P3 工程与交付

- 状态持久化从 JSON 升级为 SQLite。
  - 当前用 `app-state.json` 保存任务和登录状态摘要。
  - SQLite 更适合后续保存图片级状态、失败原因和历史查询。

- 打包前处理 Playwright 浏览器依赖。
  - 目标：普通用户不需要手动执行命令也能运行。
  - 建议：打包脚本中包含 Chromium，或首次启动自动引导安装。
